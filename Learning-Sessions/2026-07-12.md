# Topics Covered

## 1. Input

### Concepts Learned

* Difference between Input and Output.
* Information flows from the outside world into the microcontroller through input pins.
* Output sends electrical signals from the microcontroller to external hardware.
* Arduino understands electrical signals through voltage levels, while software interprets those voltage levels as information.

### Key Understanding

The Arduino does not directly understand a button press.

It observes the voltage present on an input pin and interprets it as HIGH or LOW.

---

## 2. Digital Signal

### Concepts Learned

* Digital systems work with only two logical states.
* HIGH represents logical 1.
* LOW represents logical 0.
* Voltage thresholds determine whether an input is interpreted as HIGH or LOW.
* Digital logic is robust because it ignores small electrical noise.

### Key Understanding

Voltage is the physical quantity.

HIGH and LOW are logical interpretations of that voltage.

---

## 3. Input Pin

### Concepts Learned

* Input pins observe voltage instead of generating it.
* An input pin draws extremely small current.
* The purpose of an input pin is sensing, not supplying power.
* An input pin behaves similarly to a voltmeter.

### Key Understanding

Input pins observe the electrical state of a circuit rather than driving it.

---

## 4. Floating Input

### Concepts Learned

* A floating input has no defined electrical connection.
* Floating inputs produce unpredictable readings.
* Unconnected input pins behave like tiny antennas.
* Electrical noise from the environment can change the voltage of a floating pin.

### Key Understanding

A floating input is not faulty.

Its voltage is simply undefined.

## 5. Pull-down Resistor

### Concepts Learned

* Provides a default connection to Ground.
* Prevents floating when the button is not pressed.
* Button Released → LOW
* Button Pressed → HIGH

### Working

When the button is open, the input pin remains connected to Ground through the resistor.

When the button is pressed, the input pin becomes directly connected to 5V.

---

## 6. Pull-up Resistor

### Concepts Learned

* Provides a default connection to 5V.
* Prevents floating when the button is not pressed.
* Button Released → HIGH
* Button Pressed → LOW

### Working

When the button is open, the resistor keeps the input pin connected to 5V.

When the button is pressed, the input pin becomes directly connected to Ground.

---

## 7. Internal Pull-up Resistor

### Concepts Learned

* Arduino UNO provides an internal pull-up resistor.
* Approximate value: 20kΩ–50kΩ.
* Enabled using:

```cpp
pinMode(pin, INPUT_PULLUP);
```

### Notes

* Arduino UNO provides an internal pull-up resistor.
* It does not provide an internal pull-down resistor.

---

## 8. Push Button

### Concepts Learned

* A push button is simply a switch.
* It either opens or closes a circuit.
* It does not generate electricity.
* It does not consume electricity.

### Internal Construction

Internally the push button behaves like two connected pairs of terminals.

Without pressing:

* A is internally connected to B.
* C is internally connected to D.
* No connection exists between the two pairs.

When pressed:

* All four terminals become electrically connected.

### Engineering Notes

The four pins exist primarily for:

* Mechanical stability
* Better PCB mounting
* Improved breadboard support

The additional pins do not provide additional electrical functionality.

---

## 9. Breadboard Placement

### Concepts Learned

The push button should always be placed across the center gap of the breadboard.

Otherwise, already-connected terminals may end up in the same breadboard strip, causing the switch to have no electrical effect.

---

## 10. digitalRead()

### Concepts Learned

* Reads the logical state of an input pin.
* Returns HIGH or LOW.
* Reads voltage and converts it into a digital state.

---

## 11. Engineering Mindset

A major lesson from today's session:

Instead of memorizing circuits, always ask:

1. What is connected to what right now?
2. Is there a complete path from the power source, through the components, and back to Ground?

These two questions should be used while analysing every circuit.

# Common Misconceptions Corrected

* "Open" does not mean the Arduino pin is disconnected.
* "Open" refers to the switch inside the push button.
* Pull-up and pull-down resistors do not generate HIGH or LOW.
* Their purpose is to provide a default voltage reference when no other device is driving the input.
* HIGH does not always mean the button is pressed.
* LOW does not always mean the button is released.
* The meaning depends entirely on the circuit design.
